修改UIWebView请求中的User-Agent

Date 五 26 九月 2014 Tags iOS

我希望UIWebView在发出HTTP请求时能够自定义header中的User-Agent。

首先，修改loadRequest方法的参数，是无效的:

NSMutableURLRequest *request = [NSMutableURLRequest requestWithURL:urlInstance];
[request setValue:@"my defined user-agent" forHTTPHeaderField:@"User-Agent"];
[webViewInstance loadRequest:request];

从iOS 5开始，取消了UIWebView中修改header的API，可以通过修改global user default来改变UIWebView。 我还希望在自定义User-Agent时保留默认的内容:

- (void) setGlobalUserAgent
{
    //获取默认User-Agent
    UIWebView *webView = [[UIWebView alloc] initWithFrame:CGRectZero];
    NSString *oldAgent = [webView stringByEvaluatingJavaScriptFromString:@"navigator.userAgent"];

    //添加自己的User-Agent内容
    NSString *newAgent = [NSString stringWithFormat:@"%@;%@", oldAgent, @"JingChao/250.5"];

    //设置global user default
    NSDictionary *dictionnary = [[NSDictionary alloc] initWithObjectsAndKeys:newAgent, @"UserAgent", nil];
    [[NSUserDefaults standardUserDefaults] registerDefaults:dictionnary];
}

在初始化UIWebView之前调用上面的方法即可生效。一个好的办法是，在AppDelegate的didFinishLaunchingWithOptions中调用该方法:

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions
{
    [self setGlobalUserAgent];
}

注: 代码在iOS 8.0上实验成功

网上其它解决方案

  1. 修改NSURLProtocol，有人说到这种方法会对登录页产生影响。我实验该方法失败，UIWebView发出的请求没有任何改变。
  2. 重载setValue:forHTTPHeaderField，原理我没有看明白，也没有实验。该文还提到了修改NSURLProtocol在iOS 5中是有效的，不知道上一种方法实验失败是不是因为系统版本问题。
