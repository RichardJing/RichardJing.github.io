限制UIWebView只能竖直滚动，不能水平滚动

Date 日 28 九月 2014 Tags iOS

如果网页内容超过UIWebView的边框，就会自动触发水平或者竖直的滚动功能。如果有个很“长”的页面，我希望网页内容能够与UIWebView等宽，并且只能上下滚动，不能左右滚动。

现有一个HTML页面，只输出一张300*533像素的图片:

<html>
<head>
<title>UIWebView Test</title>
<style type="text/css">
  body{margin:0px;}
</style>
</head>
<body>
  <div>
    <img src='images/ad.png'/>
  </div>
</body>
</html>

建立一个200*300的UIWebView，载入页面后是这个样子:

最终实验成功的方法是，载入URL之后，修改返回的HTML，将HTML内容缩放为匹配UIWebView的宽度，然后载入修改后的HTML。代码:

//
//  ViewController.m
//

#import "ViewController.h"

@interface ViewController () <UIWebViewDelegate>
@property (weak, nonatomic) IBOutlet UIWebView *webView;
@property (nonatomic,strong) NSURL *url;
@property (nonatomic) BOOL isLoadingFinished;
@end

@implementation ViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    self.webView.delegate = self;
    static NSString *imageURL = @"http://testsite/test.html";
    self.url = [NSURL URLWithString:imageURL];
    [self.webView loadRequest:[NSURLRequest requestWithURL:self.url]];
}

- (void) webViewDidFinishLoad:(UIWebView *)webView
{
    if (self.isLoadingFinished) {
        return;
    }

    NSString *bodyWidthStr= [webView stringByEvaluatingJavaScriptFromString: @"document.body.scrollWidth "]; //js获取body宽度
    int bodyWidth = [bodyWidthStr intValue];

    NSString *js = @"document.documentElement.innerHTML";
    NSString *html = [webView stringByEvaluatingJavaScriptFromString:js]; //获取HTML代码

    //修改HTML代码
    html = [self htmlAdjustWithPageWidth:bodyWidth
                                    html:html
                                 webView:webView];
    self.isLoadingFinished = YES;
    [self.webView loadHTMLString:html baseURL:self.url];
}

- (NSString *)htmlAdjustWithPageWidth:(CGFloat )pageWidth
                                 html:(NSString *)html
                              webView:(UIWebView *)webView
{
    NSMutableString *str = [NSMutableString stringWithString:html];
    CGFloat initialScale = webView.frame.size.width/pageWidth; //计算要缩放的比例

    //将</head>替换为meta+head
    NSString *stringForReplace = [NSString stringWithFormat:@"<meta name=\"viewport\" content=\" initial-scale=%f, minimum-scale=0.1, maximum-scale=2.0, user-scalable=yes\"></head>",initialScale];

    NSRange range =  NSMakeRange(0, str.length);
    [str replaceOccurrencesOfString:@"</head>" withString:stringForReplace options:NSLiteralSearch range:range];
    return str;
}
@end

效果如下图:

从上图可以看出，当网页内容宽度适配UIWebView宽度时，连水平滚动条都消失了。

代码中的方法在head中注入了一段meta，注入一段CSS来限制宽度应该也是可以的。

再说说我用过的几个无效方法。

调用UIWebView自身的方法都无法解决问题

首先，setScalesPageToFit方法对本问题无效。Xcode帮助文档中对这个方法的介绍:


    A Boolean value determining whether the webpage scales to fit the view and the user can change the scale.
    If YES, the webpage is scaled to fit and the user can zoom in and zoom out. If NO, user zooming is disabled. The default value is NO.

第一段后半句话好理解，可以控制用户是否能缩放，前半句“the webpage scales to fit the view ”不知道是指什么。 设置为YES之后，仍旧是之前的模样。

ScrollView没有直接解决本问题的方法。UIWebView里面有个UIScrollView用来承载网页内容，UIScrollView有几个bounce方法:

//both  vertically and horizontally
self.webView.scrollView.bounces = NO;

// only vertically
webView.scrollView.alwaysBounceVertical = NO;
webView.scrollView.alwaysBounceHorizontal = YES;

上面的代码在实验时没有改变任何东西。Xcode文档对于scrollView.bounces的解释:


A Boolean value that controls whether the scroll view bounces past the edge of content and back again.
If the value of this property is YES, the scroll view bounces when it encounters a boundary of the content. Bouncing visually indicates that scrolling has reached an edge of the content. If the value is NO, scrolling stops immediately at the content boundary without bouncing. The default value is YES.

我理解bounce只是在到达网页内容边界时给个提示，但没有看到实际的效果是什么样子。

scrollView缩放到宽度跟UIWebView边界匹配

- (void)webViewDidFinishLoad:(UIWebView *)webView
{
    CGSize contentSize = webView.scrollView.contentSize;
    CGSize webViewSize = webView.bounds.size;

    float rate = webViewSize.width / contentSize.width;

    webView.scrollView.minimumZoomScale = rate;
    webView.scrollView.maximumZoomScale = rate;
    webView.scrollView.zoomScale = rate;
}

重新运行程序，图片能够匹配宽度。虽然图片缩放了，但是scrollView.contentSize没有变化，水平依旧可以拖动，超过图片边界的地方可以看到webView的背景色。如下图:
