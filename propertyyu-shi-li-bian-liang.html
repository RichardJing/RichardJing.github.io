property与实例变量

Date 四 15 一月 2015 Tags Objective-C / iOS

当我第一次看到如下语法时，非常困惑:

@interface PersonalInfo : NSObject {
    NSString *_firstName;
}
@property NSString *firstName;
@end

firstName与_firstName长得这么像，它俩啥关系呢？

firstName是一个property, 而_firstName是一个实例变量(instance variable)。它俩是不同的两种东西，本应该没有啥关系，但是编译器从中做了一些手脚。

Objective-C定义了一个实例变量之后，必须自己写访问器来访问实例变量，如getter和setter。可能是为了体恤程序员，Objective-C提供了synthesize语法自动帮你生成getter和setter. 只要你定义了一个property就能用synthesize.

一个property背后可以有一个实例变量作为信息存放处，如果你用了如下语法:

@property NSString *firstName;
@synthesize firstName;

那么编译器除了自动生成property的getter和setter，还会帮你生成一个实例变量。实例变量默认以property的名字加前置下划线命名，即_firstName.

通过synthesize还可以将property捆绑到指定实例变量上:

@property NSString *firstName;
@synthesize firstName = otherInstanceVariable;

Objective-C更加体恤程序员的是，连synthesize都不用你做，只要你声明了一个property，编译器会自动帮你完成访问器及实例变量的synthesize操作。但是实例变量的synthesize操作是有条件的。

如果你自定义了getter或setter，那么编译器在synthesize时就不会再自动生成相应方法了。 如果你自定义了所有的访问器，编译器就不会帮你执行synthesize，也意味着跟实例变量的捆绑也不会自动完成。 对于readwrite的property, 如果你自定义了getter以及setter; 对于readonly的property你自定义了getter，那么需要手动执行synthesize.

苹果官方文档推荐使用property而不是实例变量。 使用property不但可以省去许多getter和setter，还能够通过点操作符直接调用它们，何乐而不为呢？

回到文章最初的那段代码，该作者定义了property之后，还不辞辛苦地定义了一个实例变量，是多此一举。编译器会自动帮你生成那个实例变量的。

但是在初始化方法和dealloc方法中不宜使用访问器，因为在这两个地方不确定对象是否存在，所以发送的消息可能不会成功。在这两个地方应当使用实例变量。

上述说的都是property为nonatomic情况，而property默认是atomic的。atomic访问器的实现和同步都是私有的，意味着你不能将一个同步访问器和自定义的访问器进行组合。 例如， 你对一个readwrite atomic property自定义了setter，不要指望编译帮你synthesize一个getter，你会得到编译警告。

最后再多说一下, property可以不绑定到一个实例变量上，例如一个readonly的property这样自定义了getter:

@property (readonly) NSString *firstName;
-(NSString *)firstName
{
    return @"JingChao";
}
