一个翻页图片查看器的实现

需要实现一个可翻页的图片浏览功能，就像在微信朋友圈中点开大图的效果。具体功能点如下：

  1. 在图片查看器中通过翻页显示一组图片。进入图片查看器时，可以选择打开第几张图片
  2. 有页码标识
  3. 图片适配屏幕大小并居中
  4. 可以通过双指捏合来放大/缩小图片
  5. 可以通过双击来放大图片至原始尺寸，并且将点击位置居中，再次双击则取消放大。
  6. 图片放大时，也可以翻页，但是再翻回来时，放大效果取消
  7. 翻页时，图片之间有间隙
  8. 图片内容从网络下载。为了节省流量，图片被翻到时才开始下载，下载过程中有进度条提示。第一次下载后，图片被缓存。
  9. 单击屏幕，退出图片查看器，清空图片缓存

对客户端UI的两点心得

App打开要快

本来这是很明显的事情，但经过需求的不断增改，在不知不觉中打开你的App变得慢如龟速。一个App打开不应超过2秒，否则用户不会有耐心再去探索你App有什么功能。

我遇到的情况是，App首页有很多活动宣传，活动标题、图片都要用HTTP请求获得。这些请求不能一次性都在打开页面时完成。

给用户的信息，都是他们需要的；用户需要的信息，要给全

这句话很绕，而且前后两句看起来也都是废话。

之前对于什么是“用户需要的信息”一直没有概念，现在觉得应该是“用户做出操作、选择时所需的必要信息”。 也就是说， 能够影响用户操作行为的信息，都必须清楚地告知用户；不能对用户决定造成影响的，都没必要告知用户。

git远程分支操作

如果你将一个远程库clone到本地，那么远程库默认会使用origin作为别名。可以使用git remote -v命令查看远程库的别名和信息。

远程分支merge到本地分支，以及本地分支push到远程分支的命令如下:

git pull origin master
git push origin master

如果设置本地分支追踪(track)了某个远程分支，那么就不用每次都输入"origin master"了。git clone命令本质上就是创建了本地的master分支，并追踪远程master分支。使用git remote show origin命令可以看到远程分支的追踪情况，如下:

验证车牌号是否合法

中文可以使用下面的正则表达式: 


    [\u4e00-\u9fa5]

Object-C示例代码:

- (BOOL) validateCarNo:(NSString *) carNo
{
    BOOL isValid = NO;
    if ([carNo length] != 7) {
        NSLog(@"车牌必须是7位");
    } else {
        NSString *carRegex = @"^[\u4e00-\u9fa5]{1}[A-Za-z]{1}[A-Za-z_0-9]{5}$";
        NSRange range = [carNo rangeOfString:carRegex options:NSRegularExpressionSearch];
        if (range.location != NSNotFound) {
            isValid = YES;
        }
    }
    return isValid;
}
