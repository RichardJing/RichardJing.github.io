git远程分支操作

如果你将一个远程库clone到本地，那么远程库默认会使用origin作为别名。可以使用git remote -v命令查看远程库的别名和信息。

远程分支merge到本地分支，以及本地分支push到远程分支的命令如下:

git pull origin master
git push origin master

如果设置本地分支追踪(track)了某个远程分支，那么就不用每次都输入"origin master"了。git clone命令本质上就是创建了本地的master分支，并追踪远程master分支。使用git remote show origin命令可以看到远程分支的追踪情况，如下:

$ git clone https://github.com/RichardJing/test.git
$ cd test
$ git remote show origin                                   ~/tmp/test (1)
* remote origin

将本地文件添加到github中

第1步: 创建SSH Key

你和Github之间的通信是通过SSH加密的，所以要把公钥告诉Github。 看看家目录有没有.ssh文件夹，该文件夹下有没有id_rsa和id_rsa.pub两个文件。如果有这两个文件，就可以跳到下一步了。

ssh-keygen -t rsa -C "youremail@example.com"

id_rsa是私钥，id_rsa.pub是公钥。
